## Detailed Findings

### [P0] Theme collision: dark CSS variables on light theme
- **Location:** `:root`, lines 18, 27–28
- **Category:** Theming / Accessibility
- **Impact:** `--border: rgba(255,255,255,0.12)` is white-on-white (invisible borders). `--text: #e0e0e0` applied in highlight boxes and inline on the title slide name renders near-white on white canvas. The comment still says `MOON THEME` but the link is `simple.css`. All the slide chrome (header/footer hairlines, stack card tints, split-col borders) disappears on the light substrate.
- **Recommendation:** Either (a) switch back to the moon theme and lean into the dark canvas, or (b) update every dark-mode variable to light-mode equivalents — `--border: rgba(0,0,0,0.10)`, `--text: #222222`, `--text-muted: rgba(0,0,0,0.45)`. The DESIGN.md spec says white canvas. The CSS says dark. Pick one.
- **Command:** `$impeccable colorize`

### [P1] Background orbs on title slide
- **Location:** Lines 152–154 (CSS), 284–285 (DOM)
- **Category:** Anti-Pattern
- **Impact:** `.orb-warm` (40vw × 40vw, `blur(90px)`) and `.orb-cool` (30vw × 30vw) are large smeared color blobs with zero informational value. This is the single most recognizable AI-generated slide aesthetic. The PRODUCT.md explicitly names this deck as an anti-reference to "generic AI-generated slides." Having orbs on slide 01 undermines the entire brief.
- **Recommendation:** Remove both orb elements from the DOM and delete the `.orb`, `.orb-warm`, `.orb-cool` CSS rules entirely. The title slide is strong enough without them — the typography is the hero.
- **Command:** `$impeccable distill`

---

### [P1] Identical 6-card stack grid
- **Location:** Lines 759–766 (slide 12)
- **Category:** Anti-Pattern
- **Impact:** Six cards, one template: `[emoji category] / [TOOL NAME] / [replaces X]`. This is the "identical icon-heading-text card grids" pattern the DON'Ts list calls out explicitly. Every card is structurally indistinguishable from every other card. The eye finds no anchor.
- **Recommendation:** Redesign around what each tool *replaces*, not just what it is. Replace the grid with a vertical numbered list: tool name as large display text (`clamp(1.4rem, 3vw, 2.2rem)`, Signal Blue, weight 700) followed by a single-line description. Two-column numbered list preserves the 6-item density without the cloned-card structure.
- **Command:** `$impeccable layout`

---

### [P1] Generic horizontal dot timeline (slide 04)
- **Location:** Lines 432–455 (slide 04)
- **Category:** Anti-Pattern
- **Impact:** The founder journey — the most emotionally resonant story in the deck — is rendered as five uniform dots with CSS-generated `→` arrows. The timeline label text has genuine weight (startup failure, honeymoon reset, pivoting), but the container drains it of personality.
- **Recommendation:** Drop the dot structure entirely. Treat the five beats as numbered story fragments: large step number (Signal Blue, `clamp(2rem, 5vw, 4rem)`, weight 700) left-aligned, a two-line story fragment right. Separated by hairline borders. No dots, no arrows, no symmetry between items.
- **Command:** `$impeccable layout`

---

### [P1] `split-col.before` missing Signal Blue border
- **Location:** Line 528 (slide 06), CSS at lines 208–210
- **Category:** Theming
- **Impact:** Design system rule: "A before/after split is always blue-before, teal-after." `.split-col.after` correctly has `border-color: rgba(42,161,152,0.4)`. `.split-col.before` has no class-specific color — inherits `var(--border)` (invisible on light theme). The "Before AI" column is visually dead.
- **Recommendation:** Add to stylesheet:
  ```css
  .split-col.before { border-color: rgba(66,175,250,0.3); }
  .split-col.before .split-col-label { color: var(--accent); opacity: 1; }
  ```
- **Command:** `$impeccable colorize`

---

### [P2] Bordered `accent-tag` badge on title slide
- **Location:** Lines 150 (CSS), 291 (DOM)
- **Category:** Anti-Pattern
- **Impact:** `border: 1px solid rgba(66,175,250,0.4)` on a small badge label is a textbook Canva/AI template element. Dotted-border pill chips are among the most recognizable AI UI patterns.
- **Recommendation:** Remove the border. Keep the color (`var(--accent)`) and uppercase tracking, but render as bare text — no `border`, no `border-radius`, no `padding`. A plain uppercase Signal Blue label is more confident than a framed badge.
- **Command:** `$impeccable distill`

---

### [P2] `.slide-title::after` auto-underscore on every heading
- **Location:** Lines 110–119 (CSS)
- **Category:** Anti-Pattern
- **Impact:** Every slide title automatically gets a 1.8rem × 2px Signal Blue bar via `::after`. The "heading with colored bottom line" is a common template decoration. Applied universally across 12 of 14 slides, it reads as a CSS trick, not a deliberate design choice.
- **Recommendation:** Remove the `::after` from the base `.slide-title` rule. Apply selectively only on content-heavy slides (07, 09, 11) via a `.slide-title--ruled` class.
- **Command:** `$impeccable distill`

---

### [P2] `--text-muted` renders invisible on title slide
- **Location:** Line 296
- **Category:** Accessibility
- **Impact:** `color:var(--text-muted)` = `rgba(255,255,255,0.4)` on a white canvas. At 40% white opacity over white, "Ron Zaidman · AI Israel Club" is unreadable.
- **Standard:** WCAG 2.1 AA (1.4.3, contrast ≥ 4.5:1)
- **Recommendation:** Fix at root level: `--text-muted: rgba(0,0,0,0.45)` in `:root` (part of P0 fix). No separate inline change needed once the token is corrected.
- **Command:** `$impeccable colorize`

---

### [P3] No lazy loading on images
- **Location:** Lines 340, 407, 553, 588, 622, 651, 731, 811
- **Category:** Performance
- **Impact:** All 8+ images load eagerly. Minor but trivially fixable.
- **Recommendation:** Add `loading="lazy"` to all `<img>` tags on slides 02–14 (all except the first actively-loaded slide).
- **Command:** `$impeccable optimize`

---

### [P3] Source Sans Pro not loaded
- **Location:** `<head>` (no Google Fonts import)
- **Category:** Theming
- **Impact:** DESIGN.md specifies Source Sans Pro as primary typeface. No `<link>` to Google Fonts exists. Deck falls back to Helvetica Neue / Arial.
- **Recommendation:** Either add `<link href="https://fonts.googleapis.com/css2?family=Source+Sans+3:wght@400;500;600;700&display=swap" rel="stylesheet">` to `<head>`, or update DESIGN.md to list Helvetica Neue as the intentional primary.
- **Command:** `$impeccable typeset`

## Systemic Issues

1. **Dark-mode CSS on a light theme.** Every `rgba(255,255,255,...)` value was written for a dark substrate. One fix — swapping to `rgba(0,0,0,...)` equivalents — clears 8+ issues at once: invisible chrome borders, invisible card tints, invisible muted text.

2. **Inline styles bypass the design token system.** 30+ inline `style=""` attributes override or duplicate what should be in the token system. Gap/margin utilities (`.mt-1`, `.mt-2`, `.mt-3`) are correct; inline equivalents should be replaced with those utilities.

## Positive Findings (Keep These)

- **Zigzag timeline (slide 03):** Genuinely distinctive. Three-column CSS grid with alternating left/right content. The teal final node for "Now" is semantically correct per the two-voice color rule.
- **Terminal block (slide 13):** Correct implementation. Dark surface, macOS chrome dots, monospace font, real semantic color roles. Conveys "real output" without decoration.
- **`clamp()` fluid type system:** Consistent throughout. Scales from projection to mobile with no pixel-value breakpoints.
- **`prefers-reduced-motion` media query:** Present and wired correctly.
- **`.anim` staggered entrance:** Clean expo easing, 0.1s increments, only `opacity` and `transform` animated.
- **Two-voice color semantics:** Signal Blue (active/forward) vs. Pipeline Teal (verified/complete). Consistently applied.
